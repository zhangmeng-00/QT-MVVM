# 修改日志（2026-03-24）

## 目标
- 修复 Core 框架中影响稳定性和可维护性的关键问题。
- 确保修改后程序可正常编译运行，避免引入新的功能回归。

## 本次修改内容

### 1. AppContext 连接与订阅流程重构
- `ConnectObserve` 改为幂等，避免重复连接导致重复订阅和重复分发。
- `ConnectObserve` 改为线程安全：连接动作通过 `BlockingQueuedConnection` 在 Mediator 所在线程执行。
- 连接成功后再调用 `SetupSubscriptions()`，保证订阅初始化时序正确。
- 增加 `m_connectedObservers` 与互斥保护，处理对象销毁后的清理。
- `AddComponent` 增加跨线程 `setParent` 防护，避免 Qt 跨线程父子关系错误。

涉及文件：
- `app/AppContext.h`
- `app/AppContext.cpp`

### 2. Bootstrap 生命周期安全修复
- 删除 `Logger::instance()->setParent(&ctx)`，避免静态单例被 QObject 父子机制误删。
- `SetParentIfQObject` 增加跨线程判断，跳过非法 `setParent` 并输出告警。

涉及文件：
- `app/Bootstrap.cpp`

### 3. 主窗口按钮状态绑定修复
- `btnLogin`/`btnLogout` 不再绑定同一 `loggedIn` 属性。
- 改为基于 `loggedInChanged` 的互斥启用逻辑：
  - 未登录：启用登录，禁用登出
  - 已登录：禁用登录，启用登出

涉及文件：
- `MainWindow.cpp`

### 4. SensorViewModel 事件域清理
- 删除 `ObserveData(TAG_SENSOR_TEMPERATURE)` 中跨域发布 `TAG_USER_LEVEL` 的逻辑。
- 删除运行时动态订阅 `TAG_USER_LEVEL` 的逻辑，避免重复订阅与职责耦合。

涉及文件：
- `viewmodel/SensorViewModel.cpp`

### 5. UserViewModel 初始交互修复
- `m_canPublish` 初始值改为 `true`，避免首次功能被锁死。

涉及文件：
- `viewmodel/UserViewModel.h`

### 6. ActorObserve 线程释放修复
- 析构中补全 `QThread` 对象释放路径，避免线程对象泄漏。
- 统一处理线程运行中退出与回收，析构后置空指针。

涉及文件：
- `core/actor/ActorObserve.cpp`

### 7. LogModel 线程亲和性修复
- 将数据库初始化从构造函数迁移到 `SetupSubscriptions()`。
- 保证数据库连接和使用均发生在 Actor 线程内，避免跨线程使用 `QSqlDatabase`。
- 保留初始化日志插入与日志列表首次发布逻辑。

涉及文件：
- `core/model/LogModel.cpp`

### 8. 配置结构与头文件修复
- 修复 `LoggingConfig` 中字段损坏，恢复 `databasePath`。
- 为 `ConfigLoader` 补齐 `QSettings` 与 `QDebug` 头文件依赖。

涉及文件：
- `app/config/AppConfig.h`
- `app/config/ConfigLoader.h`
